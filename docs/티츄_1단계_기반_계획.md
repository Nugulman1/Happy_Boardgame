# 티츄 1단계: 기반 (카드·상태) — 상세 계획

상위 계획: 티츄 인게임 로직 계획  
기준: [티츄 룰 정리.md](tichu/rule/티츄 룰 정리.md) §3 카드/토큰, [공통게임로직_설계.md](docs/공통게임로직_설계.md) (단계별 계획표는 "어떻게 구현할지" 정확히 명시)

**1단계 목표**: 카드/덱 정의와 게임 상태 구조만 두고, 루프·행동·점수 로직은 넣지 않는다.

---

## 1단계 세부 작업

### 1-1. 카드 타입 정의

- **산출물**: `tichu/cards.py`. **구현 메모**: [tichu/구현_메모.md](tichu/구현_메모.md)에 숫자·특수카드 정의를 기록.
- **구현 방법 (정확히)**:
  - **카드 표현**: 한 장을 하나의 타입으로 통일(예: `Card` dataclass). **숫자는 내부적으로 2~14만 사용** (J/Q/K/A 구분 없이 11,12,13,14).
  - **일반 트럼프**: `suit`(무늬 4종), `rank`(숫자 2~14). 무늬는 `Suit` enum 또는 문자열 상수 4개(예: "S", "H", "D", "C").
  - **특수 카드 4종**: 별도 타입/필드 없이 **숫자로만 구분**. 참새=**1**, 개=**20**, 봉황=**21**, 용=**22**. 참새를 1로 둔 이유는 **스트레이트에서 1-2-3-4-5처럼** 낮은 쪽으로 쓸 수 있기 때문(역스트레이트 없음). 개·봉황·용(20, 21, 22)은 일반 스트레이트 조건(2~14)에 안 걸리도록 숫자를 나눈 것.
  - **식별**: `is_special`/`get_special_kind` 같은 함수 없이, **숫자 값(1, 20, 21, 22)으로 바로 판별**.

### 1-2. 덱 구성·생성

- **산출물**: `tichu/cards.py` 내 함수.
- **구현 방법 (정확히)**:
  - **덱 생성**: `make_deck() -> list[Card]`. 무늬 4종 × (숫자 2~14) = 52장 + 참새(1)·개(20)·봉황(21)·용(22) 1장씩 = 56장. **리스트**로 반환.
  - **셔플**: 덱 생성 후 **랜덤으로 섞는 기능** 포함. 예: `shuffle_deck(deck: list[Card]) -> None` 또는 `make_deck(shuffle=True)` 등 한 번에 처리해도 됨.
  - **상태**: 수정 없음(순수 함수 또는 인자로 받은 리스트 in-place 셔플).
  - **주석**: 입력/출력·전제조건 명시.

### 1-3. 점수 매핑

- **산출물**: `tichu/cards.py` 내 함수.
- **구현 방법 (정확히)**:
  - **점수 매핑**: `def card_points(card: Card) -> int`. **if 카드 숫자 in (5, 10, 13, 21, 22)** 안에서 다시 분기:
    - **5** → 5점
    - **10**, **13**(K) → 10점
    - **21**(봉황) → -25점
    - **22**(용) → +25점  
    그 외(참새 1, 개 20 포함) → 0. 주석: 룰 §3 점수 표 참조.

### 1-4. 게임 상태(State) 구조

- **산출물**: `tichu/state.py`.
- **State = 외부만 (저장·유지)**: `GameState`에는 **팀 점수**(`team_scores`), **라운드**(`round_index`)만 둠. 게임 동안 유지되고, `init_state`는 게임 시작 시 1번만 이 필드들을 초기화.
- **라운드용 = State가 아님**: 손패(`hands`), 덱(`deck`), 선(`leader_index`), `trick_index`, `grand_tichu_declarers`는 **State에 두지 않음**. **RoundState**로 두고, **while 라운드 시작할 때마다** `new_round_state()`로 새로 만들어 준비/트릭 로직에 넘김. 내부 필드 추가·수정 시 `init_state`를 건드리지 않음.
- **팀 구분**: `team_id(i) = i % 2` → 1 또는 2. 주석: 클래스 docstring에 각 필드 의미·룰 §2·§4 참조.

### 1-5. 상태 초기화 함수

- **산출물**: `tichu/state.py` 내 함수.
- **구현 방법 (정확히)**:
  - **시그니처**: `init_state(state: GameState) -> None`. **외부 필드만** 초기화.
  - **동작**: `state.team_scores = [0, 0]`, `state.round_index = 0`. 라운드용(hands, deck 등)은 `init_state`에서 다루지 않음 — `new_round_state()`로 라운드마다 생성.
  - **주석**: 입력 `state`(수정 대상) / 출력 없음 / 수정 필드: team_scores, round_index.

### 1-6. 1단계 주석·문서 정리

- **구현 방법 (정확히)**:
  - 각 공개 함수·클래스에 docstring 또는 블록 주석으로 **입력 인자·반환값·전제조건·수정하는 상태(해당 시)** 를 적는다.
  - `cards.py`·`state.py` 모듈 상단 한 줄 요약: "1단계: 카드·덱·상태 정의. 루프/행동/점수 로직 없음."

## 완료 기준

- 덱을 한 번 생성하면 56장이고, 일반 52장 + 특수 4장으로 구분 가능.
- State를 만들면 4명 손패·덱·선·라운드/트릭·팀 점수를 담을 수 있고, 다른 모듈에서 "인자로 넘겨 수정"하는 전제가 맞음.
- 루프·행동 처리·점수 계산·종료 조건은 1단계에 포함하지 않음.

이 계획만 완료한 뒤, 2단계(준비: 배분·선언·교환·선 결정)로 이어가면 됩니다.
