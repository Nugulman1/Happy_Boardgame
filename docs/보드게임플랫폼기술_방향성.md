# 보드게임 플랫폼 기술, 방향성

# 협업 관련

1. DOCKER
2. Github
3. Todo, 일정 공유를 위한 Notion

# 필요한 기술들

1. PC 프로그램을 위한 엔진 Godot
2. 프론트, 백엔드 모두 Python으로 일단 통일 후 추후 네트워크와 게임 롤을 C++로 변경 고려
3. DB PostgreSQL

# 기능 관련 제안

1. 보드게임을 구현하는 기능부터 만드는 게 어떨까 제안
2. 티츄를 첫번째로 게임 하나 만들고 기능 천천히 추가하기 제안
3. 티츄의 룰을 구현하는 과정에서 다음과 같은 과정으로 만드는것을 제안
- 보드게임 룰 → IR 변환 → 검수 → 해당 내용대로 기능 만들기

→ 해당 과정은 어떤 보드게임을 만들던 기본 뼈대가 되는 과정임.

→ 각 턴에 할 수 있는 행동, 자원, 조건등을 미리 템플릿 만들어놓고, 거기에 코드를 추가하는 식으로 병목 발생 원천 제거 방식 좋아보임

- 디자인, UI, UX등 관련 구조에 대한 것은 추후에 추가
- 앱 개발, 커뮤니티를 위한 기술들은 추후에 추가
