# 5단계: 라운드 점수 계산 + 게임 메인 루프 연결

## 목표
- 라운드 점수를 계산해 `GameState.team_scores`에 반영한다.
- `run_game`이 준비->트릭->정산을 반복하며 1000점에서 종료되게 만든다.

## 신규 함수 (`tichu/scoring.py`)
- `calc_tichu_bonus(round_state: RoundState, finish_order: list[int]) -> list[int]`
- `calc_round_points(round_state: RoundState) -> list[int]`
- `apply_round_scores(state: GameState, round_state: RoundState) -> None`

## 점수 규칙
- 카드 점수: 5(+5), 10(+10), K(+10), 봉황(-25), 용(+25).
- 원투 발생 시:
  - 카드 점수 무시.
  - 원투 팀 +200.
  - 티츄 선언 점수(라지/스몰)만 추가 가감.
- 일반 종료(공식형) 시:
  - 각 플레이어 `captured_cards` 카드 점수 합산.
  - 마지막 플레이어 손패 카드 점수는 상대팀(1등 팀)으로 이동.
  - 마지막 플레이어가 획득했던 트릭 점수도 1등 플레이어에게 이동.
  - 팀 합산 후 `state.team_scores` 반영.

## `tichu/game_loop.py` 변경
`run_game(state)` 내부 순서:
1. `init_state(state)`
2. while not `is_game_over(state)`:
   - `round_state = new_round_state()`
   - `run_prepare_phase(round_state)`
   - `run_trick_phase(round_state)`
   - `apply_round_scores(state, round_state)`
   - `state.round_index += 1`

## 테스트 케이스
- 원투 라운드 점수(+200 + 티츄 가감) 검증.
- 일반 종료 라운드 카드점수 이동 규칙 검증.
- 봉황/용 점수 카드 반영 검증.
- 누적 1000점 도달 시 while 종료 검증.

## 단계 완료 기준
- 고정 시나리오로 1게임을 끝까지 돌렸을 때 팀 점수와 종료 시점이 규칙과 일치해야 한다.
