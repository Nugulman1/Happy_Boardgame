# 봉황 임시 설계

## 목적
- 봉황 규칙을 코드 구현 전에 별도 문서로 고정한다.
- 실제 코드 반영 전까지 `evaluate_combo` 확장 기준으로 사용한다.

## 기본 원칙
- 봉황은 일반 족보 해석 계층(`evaluate_combo`)에서 처리한다.
- 폭탄에는 사용할 수 없다.
- 한 조합 안에서 봉황은 부족한 카드 1장을 메우는 용도로만 사용한다.

## 지원 대상
- 싱글
- 페어
- 트리플
- 풀하우스
- 스트레이트
- 연속페어

## 싱글 규칙
- 봉황을 싱글로 낼 수 있다.
- 직전 싱글 위에 낼 때 비교값은 `직전 rank + 0.5`로 처리한다.
- 첫 제출 싱글일 때는 0.5로 계산한다.

## 조합별 역할
- 페어
  - 일반 카드 1장 + 봉황 1장으로 같은 rank 페어를 만들 수 있다.
- 트리플
  - 일반 카드 2장 + 봉황 1장으로 같은 rank 트리플을 만들 수 있다.
- 풀하우스
  - 트리플 쪽 부족한 1장 또는 페어 쪽 부족한 1장을 메우는 용도로 사용한다.이 때 일반 카드 2,2 + 봉황일시 더 높은 카드의 숫자 기준으로 봉황이 쓰인다.
- 스트레이트
  - 끊긴 숫자 1개를 메우는 와일드카드로 사용한다.
- 연속페어
  - pair 하나의 부족한 1장을 메우는 용도로 사용한다.

## 제외 규칙
- 폭탄 조합에는 사용 불가
- 봉황 1장으로 여러 칸을 동시에 메우는 방식은 허용하지 않음

## ComboInfo 확장 후보
