# 참새 콜 임시 설계

## 목적
- 참새 관련 규칙 중 콜 규칙만 분리해 고정한다.
- 족보 해석 계층이 아니라 상위 유효성 검사 계층에서 처리한다.

## 콜 규칙
- 참새를 낸 순간만 콜 가능
- 콜 숫자 범위는 2~14
- 콜은 라운드 종료까지 유지
- 손패에 있고 현재 조합에 포함 가능한 경우 반드시 포함해야 한다
- 콜 가능한 조합이 있으면 패스할 수 없다

## 처리 계층
- 콜은 족보 해석 계층에서 처리하지 않는다.
- 콜은 상위 유효성 검사 계층에서 처리한다.

## 이후 코드 반영 위치
- `RoundState.wish_rank`
- `set_wish_on_mahjong_play(...)`
- `must_follow_wish(...)`
- `can_play_on_trick(...)`

## 구현 메모
- `evaluate_combo(...)`는 조합 자체가 무엇인지 해석만 담당한다.
- 콜 규칙은 "지금 이 조합을 내도 되는가"를 검사하는 단계에서 반영한다.
- 패스 금지 여부도 같은 계층에서 함께 판단한다.
