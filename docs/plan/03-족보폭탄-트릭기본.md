# 3단계: 족보/폭탄 판정 + 트릭 기본

## 목표
- 카드 조합을 평가하고 덮기 가능 여부를 판정한다.
- 선 플레이어 제출 -> 후속 덮기/패스 -> 3패스 트릭 종료의 기본 루프를 완성한다.

## 상태 확장 (`tichu/state.py`)
`RoundState`에 아래 필드를 추가한다.
- `current_player_index: int`
- `trick_type: str | None`
- `trick_length: int | None`
- `last_play_cards: list[Card]`
- `last_play_player: int | None`
- `pass_streak: int`
- `trick_pile_cards: list[Card]`
- `captured_cards: list[list[Card]]`
- `finish_order: list[int]`
- `declared_tichu: set[int]`
- `wish_rank: int | None`
- `bomb_response_open: bool`

`new_round_state()` 초기화 규칙:
- `current_player_index = leader_index`
- `pass_streak = 0`
- `captured_cards = [[], [], [], []]`
- 나머지 필드는 빈값/None으로 초기화

## 신규 모듈/함수

### `tichu/hand_eval.py`
- `class ComboEval(NamedTuple):`
  - `combo_type: str`
  - `strength: tuple`
  - `length: int`
  - `is_bomb: bool`
- `evaluate_combo(cards: list[Card], wish_rank: int | None = None) -> ComboEval | None`
- `can_beat(prev: ComboEval, new: ComboEval) -> bool`

판정 규칙:
- 지원 족보: 싱글, 페어, 연속페어, 트리플, 풀하우스, 스트레이트, 폭탄(포카드/스트플).
- 같은 종류 + 같은 장수에서만 일반 덮기 허용.
- 폭탄은 일반 족보 위에 덮기 가능, 폭탄끼리는 강도 비교.
- 백스트레이트 없음.
- 일반 플러시 족보 없음.

### `tichu/actions.py`
- `can_play_on_trick(round_state: RoundState, player: int, cards: list[Card]) -> bool`
- `apply_play(round_state: RoundState, player: int, cards: list[Card]) -> None`
- `apply_pass(round_state: RoundState, player: int) -> None`
- `end_trick(round_state: RoundState) -> None`

행동 처리 규칙:
- 선 플레이어는 유효 조합이면 제출 가능.
- 후속 플레이어는 덮기 가능 시에만 제출 가능.
- `apply_play`는 손패 제거, `last_play_*` 갱신, `pass_streak=0`, 트릭 타입/길이 갱신.
- `apply_pass`는 `pass_streak += 1`.
- `pass_streak == 3`이면 `end_trick` 실행.
- `end_trick`는 `trick_pile_cards`를 획득자 `captured_cards[winner]`로 이동.
- 트릭 획득자가 이미 손패 0장이면 다음 유효 플레이어가 선.

## 테스트 케이스
- 족보 판정 성공/실패(각 족보 1개 이상).
- 덮기 성공/실패(종류/길이/강도 불일치 포함).
- 폭탄 vs 일반/폭탄 비교.
- 3패스 시 트릭 종료와 획득 카드 이동.

## 단계 완료 기준
- `run_prepare_phase` 이후 트릭 1개를 규칙대로 완료할 수 있어야 한다.
