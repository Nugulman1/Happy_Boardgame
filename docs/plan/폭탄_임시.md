# 폭탄 임시 설계

## 목적
- 폭탄 규칙을 문서로 먼저 고정한다.
- 구현 시 `evaluate_combo`, `can_beat`, 상위 액션 계층 반영 기준으로 사용한다.

## 폭탄 종류
- 포카드
- 스트레이트 플러시

## 기본 흐름
- 폭탄은 트릭이 활성 상태일 때 누구나 즉시 제출할 수 있다.
- 폭탄은 현재 턴과 무관하게 인터럽트 액션으로 처리한다.
- 동시 폭탄은 서버 도착 순서 우선으로 처리한다.

## 비교 규칙
- 일반 족보 위에 폭탄으로 덮을 수 있다.
- 폭탄끼리는 폭탄끼리 비교한다.
- 포카드 < 스트레이트 플러시
- 같은 종류 폭탄끼리는 strength 비교로 처리한다.

## 제한
- 봉황 포함 폭탄 불가
- 개 사용 중 폭탄 불가
- 트릭 종료 후 폭탄 불가

## 이후 코드 반영 위치
- `tichu/combo_info.py`
  - `evaluate_combo(...)`에 폭탄 판정 추가
  - `can_beat(...)`에 폭탄 예외 비교 추가
- 상위 액션 계층
  - `play_bomb(...)` 추가

## 구현 메모
- 응답 단계 방식이 아니라 즉시 인터럽트 액션 구조로 구현한다.
- 프론트와 무관하게 백엔드 기준에서 폭탄 제출이 곧 상태 변경 이벤트가 되도록 설계한다.
