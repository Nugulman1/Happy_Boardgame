- Godot 메인 화면을 `상단 상태 바 / 좌측 유틸리티 / 중앙 플레이 스테이지 / 우측 컨텍스트` 구조로 재배치했다.
- 로비와 direct/dev 컨트롤은 항상 보이게 유지하고, 인게임 테이블과 손패를 중앙 우선 영역으로 옮겼다.
- 버튼 비활성화 사유와 현재 상태를 `Room / Action / Exchange / Selection / Hand` 힌트 라벨로 분리해 표시하도록 정리했다.
- 플레이어 패널은 턴 강조, viewer 표시, OUT 상태 구분이 더 잘 보이도록 조정했다.
- 최근 effect와 table motion 텍스트는 좌측 보조 영역으로 이동시켰다.
- 작은 창에서 잘리는 경우를 대비해 루트 레이아웃을 `ScrollContainer`로 바꾸고, project stretch 설정과 폭 기준 압축 레이아웃을 추가했다.
- 중앙 테이블 카드를 단순 텍스트 대신 색이 들어간 카드 칩 형태로 바꾸고, 손패 버튼도 선택 상태가 더 잘 보이도록 스타일링했다.
- 초상화 placeholder를 `BASE / DECLARED / FAILED SLOT` 형태의 자산-ready 슬롯으로 정리하고, 상태별 asset slot tooltip을 추가했다.
- 선언 배너와 결과 오버레이의 색과 문구를 placeholder 단계에서 더 고정된 톤으로 정리했다.

## 레이아웃 / 상태 바

- 초기 빈 상태
  - 기대값: `Mode: waiting for room or direct game`, 빈 테이블/손패, 플레이어 4칸 placeholder 표시

- 상태 바 정보
  - 기대값: 서버 상태, 라이브 소켓 상태, phase, turn, 점수/round 정보가 한 영역에 정리되어 보임

- 오류 메시지
  - 기대값: 잘못된 액션이나 연결 실패 시 상단 오류 라벨에 바로 표시됨

- 선언 안내
  - 기대값: 평상시에는 상단 `Announcement` 라벨에 placeholder 또는 최근 안내가 보이고, 선언 시 오버레이 배너도 함께 표시됨

- 작은 창 스크롤
  - 기대값: 창 높이나 폭이 작아도 루트 화면이 잘려서 사라지지 않고 스크롤로 아래 영역에 접근 가능

- 반응형 압축
  - 기대값: 폭이 좁아지면 플레이어 그리드, 좌석 버튼, dev 버튼 그리드가 더 촘촘한 배치로 바뀜

## 로비 / 멀티플레이

- 방 생성
  - 기대값: room code와 내 seat 정보가 좌측 로비 카드에 반영됨

- 좌석 점유
  - 기대값: 좌석 버튼이 `open`, `claimed`, `connected`, `me` 상태로 구분되어 보임

- 시작 가능 상태
  - 기대값: 4인 조건 충족 전에는 `Start` 비활성, 충족 시 활성

- 로비 힌트
  - 기대값: 현재 로비 상황에 맞는 안내 문구가 `RoomHintLabel`에 표시됨

- 세션 복구
  - 기대값: 저장된 room/session이 있으면 `Mode: restoring...`과 복구 안내가 먼저 보임

## 플레이 스테이지 / 액션 피드백

- 인게임 전환
  - 기대값: room start 이후 중앙 플레이 스테이지가 메인 시선 영역으로 동작함

- 손패 선택 피드백
  - 기대값: 선택한 카드가 강조되고, hand/selection 힌트가 현재 선택 가능 여부를 설명함

- viewer/actor 불일치
  - 기대값: 손패 선택이 막히고 `Viewer and Actor`를 맞추라는 안내가 보임

- 교환 단계
  - 기대값: `Exchange hint`가 0/3, 1/3, 2/3, ready 상태로 바뀜

- 트릭 단계
  - 기대값: `Selection hint`에 preview 결과나 선택 부족 이유가 표시됨

- 소켓 미연결
  - 기대값: 액션 버튼이 비활성화되고 `live socket required` 계열 안내가 보임

- 드래곤 수령자
  - 기대값: 가능 시 우측 액션 컨텍스트에만 선택 버튼이 나타남

- 테이블 카드 칩
  - 기대값: 중앙 테이블 카드는 `(empty)` 텍스트만이 아니라 카드가 있을 때 수트별 색이 들어간 칩 형태로 표시됨

- 손패 카드 스타일
  - 기대값: 손패 버튼은 기본/선택 상태의 색 차이가 분명하고, 선택 시 tooltip과 강조색이 적용됨

## 플레이어 / 결과 / 연출

- 플레이어 4칸 표시
  - 기대값: 각 칸에 이름, 손패 수, 상태, 배지, 초상화 placeholder 표시

- 초상화 슬롯 명시
  - 기대값: 각 플레이어 초상화 영역에 `BASE SLOT`, `DECLARED SLOT`, `FAILED SLOT` 계열 상태가 표시되고, tooltip에 asset slot 이름이 보임

- 현재 viewer 표시
  - 기대값: 현재 viewer 플레이어 칸 이름에 `(viewer)`가 붙고 패널 톤이 다르게 보임

- 턴 강조
  - 기대값: 현재 턴 플레이어 칸만 하이라이트

- OUT 상태
  - 기대값: 손패를 비운 플레이어는 `OUT`으로 보이고 패널 톤이 달라짐

- 선언 배너 / 초상화 상태
  - 기대값: `Grand Tichu`, `Small Tichu` 시 상단 배너가 고정 톤 문구로 표시되고, `Declared` 초상화 상태가 반영됨

- 실패 초상화
  - 기대값: `round_finished`에서 선언 실패자만 `Failed` 상태를 잠깐 보여줌

- 라운드 / 게임 결과
  - 기대값: 우측 결과 영역과 중앙 오버레이가 함께 갱신되고, 오버레이가 사라진 뒤에도 점수 요약 텍스트는 남음

- 결과 패널 톤
  - 기대값: 라운드 종료와 게임 종료 오버레이가 동일한 placeholder가 아니라 서로 다른 톤과 문구 차이를 가짐

- 최근 effect / table motion
  - 기대값: 좌측 보조 영역에서 최근 effect 로그와 테이블 갱신 문구를 확인할 수 있음
