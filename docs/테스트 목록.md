# 테스트 목록

## 플레이어 4칸 UI

- 도커 서버 확인
  - `docker compose ps`
  - `curl http://localhost:8000/health`
  - `docker compose exec app sh -lc 'cd /app && PYTHONPATH=/app pytest app/test_tichu_api.py'`
  - 기대값: `/health` 정상, 테스트 통과

- Godot 씬 로드
  - [godot/scenes/Main.tscn](/home/nugulman/dev/Happy_Boardgame/godot/scenes/Main.tscn) 열기
  - 기대값: 에러 없이 로드, `Players` 영역에 4칸 표시

- 초기 빈 상태
  - 기대값: 각 칸에 `Player ?`, `Hand: -`, `Status: -`, `Badge: -`

- 게임 생성 후 플레이어 표시
  - 기대값: `Player 0~3`이 각 칸에 맞게 표시되고 손패 수가 들어감

- 턴 강조
  - 기대값: 현재 턴 플레이어 칸만 하이라이트

- 선언 배지
  - 기대값: 선언 시 `GRAND`, `SMALL`, 둘 다면 `GRAND, SMALL`

- 상태 표시
  - 기대값: 진행 중은 `ACTIVE`, 손패를 다 비우면 `OUT`

- 재접속 복구
  - 기대값: 재접속 후 4칸 정보, 턴 강조, 선언 상태 복구

- 멀티플레이 확인
  - 기대값: 여러 클라이언트에서 같은 플레이어 상태가 동일하게 보임

## 선언/점수 패널

- 선언 배너
  - 기대값: `Grand Tichu`, `Small Tichu` 시 상단 배너 표시 후 자동으로 사라짐

- `No Grand Tichu`
  - 기대값: `declare=false`일 때 큰 선언 배너가 뜨지 않음

- 라운드 결과 패널
  - 기대값: `round_finished` 시 중앙 결과 패널 표시 후 자동으로 사라짐

- 게임 종료 패널
  - 기대값: `game_finished` 시 최종 결과 패널 표시 후 자동으로 사라짐

- 점수 요약 유지
  - 기대값: 결과 패널이 사라진 뒤에도 `Round Score` 요약 텍스트는 남음

- 초상화 placeholder
  - 기대값: 각 플레이어 칸에 `Portrait` 자리 표시가 보임

## 초상화 3상태 / 중앙 카드 갱신

- 초상화 기본 상태
  - 기대값: 초기 로드 시 각 플레이어 초상화가 `Base` 상태로 보임

- 선언 초상화
  - 기대값: `Grand Tichu`, `Small Tichu` 시 해당 플레이어만 `Declared` 상태로 바뀌고 잠시 후 `Base`로 복귀

- 실패 초상화
  - 기대값: `round_finished`에서 실패한 선언자만 `Failed` 상태로 바뀌고 잠시 후 `Base`로 복귀

- `No Grand Tichu`
  - 기대값: `declare=false`일 때 초상화 상태가 바뀌지 않음

- 카드 중앙 갱신
  - 기대값: 카드 플레이 시 카드가 날아오지 않고 중앙 카드 영역이 자연스럽게 갱신되며 짧게 강조됨
