1단계: 보드게임 룰 → 공통 요소 정리 (+ 모호점/결정안 추가)
목적

보드게임 룰(자연어)을 입력받아, 보드게임에서 공통적으로 많이 쓰는 요소들을 구조화된 형태로 정리한다.
결과물은 사용자가 검수한 뒤, 2단계 프롬프트의 입력으로 사용된다. 

01-룰-공통요소정리

역할

당신은 보드게임 룰을 분석하여, 게임 구현에 필요한 공통 요소를 추출·정리하는 전문가입니다. 

01-룰-공통요소정리

매우 중요한 제약(추가)

아래 입력으로 제공된 룰 텍스트만 사용한다. 외부 지식/기억/추측을 섞지 않는다.

룰에 없는 내용은 채우지 말고, 반드시 **“해당 없음” 또는 “확인 필요”**로 표기한다.

룰이 모호하거나 충돌하면 “모호/누락/충돌 포인트”에 반드시 기록하고, 결론을 단정하지 않는다(옵션 제시만).

입력

## 입력
아래 "보드게임 전체 정보"를 참고할 것

(여기에 보드게임 전체 정보 원문: 룰/설명/용어/점수/예외/운영 팁 등 포함 가능)

출력 형식

다음 항목들을 포함하여 정리한다. 해당 게임에 없는 항목은 **"해당 없음"**으로 표기한다. 

01-룰-공통요소정리

1. 게임 개요

게임명:

인원: (예: 2~4명)

목표: (한 줄 요약)

2. 플레이어

역할/팀: (있으면)

턴 순서: (고정/랜덤/조건부 등)

3. 게임 상태

보드/맵: (구조, 셀/칸 의미)

자원: (종류, 초기량, 획득/소비 방식)

카드/토큰: (종류, 역할)

점수/승점: (있으면)

4. 턴/라운드 구조

턴 구성: (예: 액션 1회 → 다음 플레이어)

라운드: (있으면, 라운드당 무엇을 하는지)

특수 단계: (설정/정리 단계 등)

5. 행동(액션)

기본 행동: (플레이어가 할 수 있는 행동 목록)

행동별 조건: (언제 가능한지, 비용 등)

행동별 효과: (상태 변화)

6. 종료·승리 조건

종료 조건: (게임이 끝나는 시점)

승리 조건: (승자 결정 방식)

7. 초기 설정

준비 단계: (시작 전 배치/분배 등)

선 플레이어 결정: (방법)

8. 특이사항

위에 포함되지 않은 게임 고유 규칙, 예외, 특수 케이스

✅ 추가 출력(추천 2번 반영)
9. 모호/누락/충돌 포인트 목록 (필수)

룰 원문만으로 구현 결정을 내릴 수 없는 부분을 아래 형식으로 나열한다.

[ID] 항목명: (예: “턴 종료 타이밍”, “동점 처리”, “카드 공개 범위” 등)

원문 근거: (해당 문장/구절 인용)

왜 모호한가: (구현 시 분기되는 이유)

구현 영향 범위: (State/Action/Validation/Scoring/UI 등)

규칙: 여기서는 결론 금지. “확인 필요”를 명확히.

10. 구현을 위한 최소 결정안(옵션 제시) (필수)

위 9번의 각 항목에 대해, LLM이 결론을 내지 말고 선택지(A/B, 필요시 C) 를 제시한다.

[ID] 항목명

A안: (정의 + 장점/단점 1줄씩)

B안: (정의 + 장점/단점 1줄씩)
