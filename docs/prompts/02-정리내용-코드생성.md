# 2단계: 정리된 룰 → 게임 플러그인 코드 생성

## 목적

1단계에서 정리·검수된 보드게임 공통 요소를 입력받아, **게임 플러그인(Python) 코드**를 생성한다.  
생성된 코드는 공통 엔진에 주입되어 실행된다.

---

## 역할

당신은 보드게임 룰 정리본을 입력받아, Python으로 게임 플러그인을 구현하는 전문가입니다.  
공통 엔진이 제공하는 인터페이스에 맞춰 플러그인만 작성합니다.

---

## 입력

아래 `{{RULES_SUMMARY}}` 자리에 **1단계에서 정리·검수된 내용**을 넣어 사용한다.

```
{{RULES_SUMMARY}}
```

---

## 공통 엔진이 담당 (플러그인에서 하지 않음)

- 게임 상태(State) 저장/관리
- 플레이어/턴 관리(기본 구조)
- 액션 수신(WebSocket) 및 브로드캐스트
- 이벤트 로그(리플레이/디버깅)
- 테스트 러너(샘플 시나리오 실행)

---

## 플러그인이 담당 (구현 대상)

- **게임 규칙**: 액션 검증, 상태 적용
- **상태 초기화**: 게임 시작 시 보드/카드/자원 배치
- **종료/승리 조건**: 게임 종료 여부, 승자 판정
- **게임 고유 액션 정의**: 어떤 액션을 받을지, 파라미터 구조

---

## 출력 형식

### 1. 플러그인 코드 (Python)

- 공통 엔진의 플러그인 인터페이스에 맞는 Python 모듈
- 파일명: `plugin.py` (또는 `games/{game_id}/plugin.py`)

### 2. 게임 설정 (YAML/JSON)

- `games/{game_id}/game.yaml` 형식
- 게임 메타정보, 초기 설정값, 액션 스키마 등

### 3. 사용 설명

- 생성된 코드의 구조 요약
- 공통 엔진과의 연동 포인트
- 테스트 시나리오 예시 (선택)

## 플러그인 인터페이스 (예상)

> ※ 실제 엔진 구현 시 구체화됨. 아래는 초안.

```python
# 플러그인은 다음을 구현:
# - init_state() -> dict  : 초기 게임 상태
# - validate_action(state, player_id, action) -> bool
# - apply_action(state, player_id, action) -> dict  : 새 상태 반환
# - check_win(state) -> str | None  : 승자 ID 또는 None
# - check_end(state) -> bool
```

```
### 5. 행동(액션)
- 기본 행동: 덱에서 뽑기, 버린 카드 가져오기, 카드 버리기
...
```

## 사용 방법

1. `{{RULES_SUMMARY}}`를 1단계 검수 완료된 정리 내용으로 치환
2. LLM에 프롬프트 전달
3. 생성된 코드를 `games/{game_id}/` 에 저장
4. 공통 엔진으로 로딩 후 실행·테스트

---

## 주의사항

- **공통 엔진은 수정하지 않음** — 플러그인 영역만 구현
- 인터페이스는 엔진 구현 시 확정되며, 그때 이 프롬프트도 함께 수정 필요
